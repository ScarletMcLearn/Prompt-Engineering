Adding New Information to a Prompt: Expanding the Model’s Capabilities

One of the most powerful things we can do in a prompt is provide a language model with information it does not already have.

Large Language Models are trained on data available up to a specific point in time. After that, there is a so-called “cut-off date.” Events, data, or documents created after that point are not part of the model’s training. In addition, the model does not know anything that was not included in its training data.

Modern models are much more capable than earlier versions. Some versions can browse the web, access external tools, or work with uploaded files and databases. However, this does not mean the model “knows everything.”

Even if a model can search online, it may not have access to the specific information that matters to you — especially in business contexts.

For example, organizations often work with private data such as:

  • Internal documents

  • Corporate databases

  • Reports

  • Financial data

  • Customer records

If you want the model to reason using that information, you must provide it directly in the prompt.

Example: Adding Missing Data

Earlier, we asked:

“How many birds are currently outside my house?”

The model cannot answer this because it has no real-time access to your physical environment. It does not know where your house is, and it does not have access to cameras or sensors.

So it responds with something like:

“As a language model, I do not have the ability to perceive the physical world.”

In other words: it lacks data.

The solution is simple:
Provide the missing data directly in the prompt.

For example:

Historical observations of the average number of birds outside my house on a typical day:
January — 120
February — 150
March — 210
April — 408

It is currently March. Based on these observations, estimate how many birds are outside my house.

Now the model has relevant information. It can reason based on the provided data.

It may respond:

“Based on the historical observations, the average number of birds in March is approximately 210.”

Previously, the model could not answer. Now it can.

Universal Principle

This approach works with any type of data.

If you have private documents, you can insert them into the prompt and write:

“Below is the relevant information. Using only this material, answer the following question…”

The model will then reason based strictly on the provided content.

However, you must consider confidentiality.
If you are using an external service, you are sending data to its servers. Always evaluate whether this is acceptable. If you use a local model, the security considerations are different.

Modern AI Architecture (RAG)

Most modern AI systems in 2026 follow this general workflow:

  1. Search for relevant documents (internally or online).

  2. Select the most relevant fragments.

  3. Insert them into the prompt.

  4. Ask the model to generate an answer based on those fragments.

The model is not retrained each time.
Instead, we expand its context dynamically.

This approach is known as RAG (Retrieval-Augmented Generation) — generation enhanced by retrieval.

In other words:
The model does not answer purely “from memory.” It answers based on retrieved and injected data.

This is how many corporate AI assistants, legal tools, internal knowledge systems, and enterprise search applications are built today.

Why Explicit Rules Matter

The model does not know your hidden assumptions.

If your task involves special conditions, you must state them explicitly.

Example:

My house is under a sealed glass dome.
No animals can enter or leave.
All animals inside live forever.

Historical observations:
January — 120
February — 120
March — 120
April — 120

It is March.
How many birds are outside my house?

Now the reasoning changes completely.

The model will conclude:

“Since no animals can enter or leave and all animals live indefinitely, the number remains constant at 120.”

In reality, this scenario is impossible.
But within the defined rules, the reasoning is logically consistent.

If unusual assumptions are not explicitly stated, the model will default to common-sense world knowledge.
