Cognitive Verification Pattern

One interesting property of large language models is that, in many cases, they reason more effectively when a task is broken down into smaller sub-tasks. Sometimes answer quality improves simply because the model is asked to explain its reasoning step by step.

Why does this happen?

There are several hypotheses. Intuitively, we can assume the following: during training, the model frequently encountered complex problems that were decomposed into sequential steps. When it follows a step-by-step structure, it becomes easier for the model to predict the next logical fragment of reasoning and arrive at a more accurate conclusion.

Later in the course, we will discuss task decomposition strategies in more detail. For now, let’s focus on a simple and highly practical technique: the Cognitive Verification Pattern.

What is the core idea?

The idea is straightforward. We instruct the model so that when it receives a question, it:

  1. First generates several additional sub-questions that help clarify the original question.

  2. Then answers those sub-questions.

  3. Finally combines those answers into a structured, well-supported final response.

In other words, the model does not answer immediately. It first “verifies” the question by breaking it into components.

Why does this work?

If we ask a broad or abstract question, the model may respond superficially.

But if we require it to identify related factors, recall typical variables, and structure the problem, the model gains richer context for reasoning. This is also valuable for us as users: we begin to see additional dimensions of the problem that we might not have considered.

Example

We start by instructing the model:

When you receive a question, follow these rules:

  1. Generate several additional questions that would help answer the original question more accurately.

  2. Combine the answers into a final response.

The model confirms it will follow these rules.

Now we ask:

How many mosquitoes live in my backyard?

This is vague and difficult to answer directly.

Using the Cognitive Verification Pattern, the model first generates clarifying questions:

  • What is the size of your yard?

  • What climate zone are you in?

  • What time of year is it?

  • Is there standing water nearby?

  • Are there plants that attract mosquitoes?

Notice how the model automatically identifies relevant ecological factors.

We respond:

  • The yard is about 3,000 square meters

  • Located in a temperate climate

  • It is June

  • No ponds, but there is a decorative fountain with circulating water

  • Plenty of grass, shrubs, and trees

  • The yard is well-maintained and treated for insects regularly

Now the model forms a final answer based on climate, seasonality (June being peak mosquito activity), vegetation, water presence, and pest control.

Instead of producing a random number, it may estimate a range — for example, dozens to a few hundred during peak season, likely toward the lower end due to regular treatment.

Why is this useful?

Even if the estimate is imperfect (facts should always be verified), we receive:

  • Structured analysis

  • Key influencing factors

  • A range instead of a guess

  • Transparent reasoning

This increases trust and encourages structured thinking.

Format of the Cognitive Verification Pattern

To apply this pattern, the prompt should include these fundamental contextual instructions:

  • When you receive a question, follow these rules.

  • Generate several clarifying sub-questions that would help answer the original question more accurately.

  • Answer each sub-question.

  • Combine the results into a final response to the main question.

The core idea: break the problem into parts first, analyze sequentially, then synthesize.

Adapted Examples (Culturally Neutral)

1. Recipe Planning

When asked to create a recipe:

  • Generate clarifying questions about available ingredients and kitchen equipment.

  • Use the answers to propose a realistic recipe.

2. Travel Planning

When asked to plan a trip:

  • Generate clarifying questions about budget, preferred activities, and transportation.

  • Use the answers to build a more accurate itinerary.
