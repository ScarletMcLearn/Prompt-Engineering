Outline Expansion Pattern

Large language models have fundamental limitations. There is always a limit to:

  • how much input (prompt + conversation context) they can process at once

  • how much output they can generate in a single response

Models are improving. Context windows are getting larger. Output length is increasing. But limitations will always exist.

For example, if you tell a model:
“Write an entire book.”

It cannot generate a full-length book in one response. At some point, it must stop.

Similarly, you cannot upload tens of thousands of pages and expect the model to process everything in a single pass. There will always be boundaries.

Working in Parts

To handle large tasks, we need a structured approach.

Instead of generating everything at once, we:

  1. Create a structured outline

  2. Gradually expand it

  3. Generate content section by section

  4. Assemble everything into a coherent whole

This method works not only for writing books. It applies to:

  • course creation

  • technical documentation

  • software architecture

  • research papers

  • business strategy documents

  • any hierarchical project

If a task can be represented as a structured hierarchy, this pattern works extremely well.

That is the core idea of the Outline Expansion Pattern.

Why Structure Is Powerful

Structure allows us to:

  • Copy individual sections back into the prompt

  • Specify exactly where we are in the document

  • Maintain logical consistency

  • Control generation step by step

For example, we can say:

“Expand section 3.2. Do not repeat earlier sections. If needed, reference them briefly.”

We can also:

  • Refer back to previous sections

  • Mention future sections that will be developed later

  • Keep track of dependencies between parts

This is particularly useful when working under token limits.

When This Pattern May Not Work

This approach works best when a task naturally fits into a hierarchical structure.

However, some highly interconnected projects do not divide cleanly into structured layers. In such cases, this method may be less effective.

Still, for most structured content and project work, it is extremely powerful.

How the Pattern Works

A typical instruction might look like this:

Act as an outline expander.
Generate a bullet-point outline based on the input I provide.
Then ask which section should be expanded.
When I select a section, create a more detailed sub-outline for it.
After expanding, ask again which section should be expanded next.

Notice that this combines multiple patterns:

  • Persona Pattern (“Act as…”)

  • Ask for Input Pattern (“Ask which section to expand”)

  • Controlled generation flow

Example

Let’s say we ask:

Create an outline on the topic: “How to Write Effective Prompts.”

The model might generate:

  1. Introduction

  2. Understanding How Models Work

  3. Writing Effective Prompts

  4. Common Mistakes

  5. Conclusion

Then we say:

Expand section 3.

The model generates:

  1. Writing Effective Prompts
    3.1 Be explicit about the task
    3.2 Specify output format
    3.3 Provide sufficient context
    3.4 Use examples

Then we say:

Expand 3.2.

And it creates an even more detailed substructure.

This step-by-step expansion allows us to:

  • Generate content in manageable pieces

  • Always know where content belongs

  • Maintain coherence across large documents

The Core Advantage

Because of model length limitations:

  • We generate parts

  • We understand their place in the system

  • We reinsert relevant sections when needed

  • We maintain structural consistency

This is how you build large systems using language models despite context limitations.

Important Nuance

When working in parts, a key challenge emerges:

How do we ensure that each fragment:

  • Is self-contained

  • Does not contradict earlier sections

  • Remains aligned with future sections

Sometimes you must manually paste relevant structural context back into the prompt so the model does not “forget” previous decisions.

This is an unavoidable complexity when building large systems.

The Outline Expansion Pattern gives you the ability to:

  • Move up and down the hierarchy

  • Expand high-level ideas

  • Dive into details

  • Return to higher levels

  • Maintain a global map of the project

This is how complex systems can be built under model constraints.

Outline Expansion Pattern – Format

To use this pattern, your prompt should include these core instructions:

  • Act as an outline expander.

  • Generate a numbered outline based on the topic I provide.

  • Each section may contain no more than 3–5 subpoints.

  • Ask which section should be expanded.

  • When I choose a section, generate a deeper sub-outline.

  • After expanding, ask again which section to expand next.

  • Ask me which topic the outline should be created for.

Example Prompt

Act as an outline expander. Generate a numbered outline based on the topic I provide. Each section may contain no more than 3–5 subpoints. After generating the outline, ask which section should be expanded. When I choose one, create a deeper structured outline for that section. Then ask again which section should be expanded next. Ask me which topic to begin with.
