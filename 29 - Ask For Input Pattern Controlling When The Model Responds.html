Ask for Input Pattern

When working with large language models, we often describe a set of rules in the prompt that the model must follow. These rules shape the output and essentially “program” the model’s behavior.

However, a practical problem appears:

How do we prevent the model from immediately applying all the rules and generating a long response?

For example, if we use the inverted interaction pattern and say, “Ask me questions,” the model may immediately generate 10–20 questions at once. In some scenarios, that is not what we want.

This is where the Ask for Input Pattern becomes useful.

Core Idea

The idea is simple:

You describe the rules first.
Then you explicitly tell the model to request the next input from you.

This forces the model to stop and wait instead of continuing automatically.

Pattern Format

To use this pattern, your prompt should include a clear instruction such as:

Ask me to provide X.

Replace X with the specific type of input, for example:

  • “the first task”

  • “a question”

  • “a goal”

  • “an ingredient”

  • “a document”

  • “a message thread”

Example 1 (Adapted to International Context)

From now on, I will paste email threads into our conversation.
You must summarize the position of each participant.
Present the result as a structured bullet list.
At the end, list all open questions or tasks directly addressed to me.
Ask me for the first email thread.

What happens?

Instead of inventing a sample email thread, the model replies:

Please provide the first email thread.

It stops and waits.

Example 2 (Neutral & Creative)

From now on, translate everything I write into a series of dog sounds and actions that reflect the dog's reaction.
Ask me for the first sentence to translate.

Again, the model waits for the input instead of starting on its own.

Why This Matters

Without this pattern, the model may:

  • invent tasks on its own

  • generate unnecessary text

  • take control of the interaction

  • “pollute” the dialogue with assumptions

This becomes especially problematic in interactive systems or structured workflows.

Example Without the Pattern

Suppose you write:

Every time I ask you to write a prompt for a task, first list alternative approaches and then write a prompt for each approach.

If you stop here, the model might respond:

Here are three alternative approaches for organizing a messy desk...

But you never asked for that task.
The model invented it.

The Simple Fix

Add one final sentence:

Ask me for the first task.

Now the logic changes completely.

Instead of generating content, the model responds:

What task would you like to start with?

It transfers control back to you.

Why This Works

Large language models predict the next piece of text.

When your prompt ends with:

Ask me for the first input.

the model interprets that as the next required action.
Instead of continuing with content generation, it switches to request mode.

This technique is surprisingly reliable.

Sometimes a single sentence at the end of your prompt is enough to prevent the model from “thinking ahead” and taking over the interaction.
