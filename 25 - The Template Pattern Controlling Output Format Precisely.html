The Template Pattern

Getting a language model to produce output in exactly the format you need is often challenging.

The Template Pattern helps you enforce strict formatting and clearly specify where generated information should appear.

The idea is simple:
You provide a template — the model fills it in.

It works like instructing an assistant:

“Here is the template I use. These are placeholders. Fill them in.”

The model takes the structure and inserts content in the correct positions.

That is the essence of the Template Pattern.

Core Idea

We define the output structure in advance, and the model must follow it.

Yes, similar effects can be achieved using few-shot examples.
But here we explicitly use a structural template approach.

We begin with a clear instruction:

I am going to provide a template for your output.

Then we clarify:

  • Words written in ALL CAPS are placeholders.

  • Replace the placeholders with your output.

  • Preserve the overall formatting and structure.

This prevents the model from “reformatting creatively.”

Example with Markdown Formatting

Suppose the template looks like this:

Question QUESTION
Answer ANSWER

Here:

  • “Question” is fixed text (part of the structure).

  • “QUESTION” is a placeholder.

  • Same with “Answer” and “ANSWER.”

Then we say:

In the next message, I will provide text. Create 20 questions using my template.

We provide a source text (for example, an encyclopedia article about early human migration).

The model generates 20 structured blocks:

  • Question heading

  • Question

  • Answer heading

  • Answer

The structure is preserved exactly.

What Matters Here

The placeholder “QUESTION” is not just empty space.
The model interprets it as an instruction to generate a question.

The model:

  • Extracts relevant content

  • Generates structured questions

  • Inserts them strictly into the template

This is not mere formatting — it is structured reasoning within defined constraints.

Professional Use Case

Imagine you are preparing for a first meeting with a colleague — for example, a technology executive or a cybersecurity expert.

You review public information: interviews, company website, articles.

Then you ask:

Create 20 preparation questions using my template.

If precision matters, you add:

Use only the provided text.
Do not add information that is not present.

Now the model works strictly within the supplied context.

Smart Placeholders

Placeholders can define:

  • Length (ONE_SENTENCE, 200_WORDS)

  • Structure (TABLE, BULLET_LIST, PARAGRAPH)

  • Style (FORMAL, ANALYTICAL)

  • Focus (EMPHASIZE_RISKS, HIGHLIGHT_ACHIEVEMENTS)

Example template:

Name: NAME

Short Summary: ONE_SENTENCE

Detailed Description: ONE_PARAGRAPH

If you provide a biography and say:

Extract all mentioned individuals and fill the template for each.

The model will:

  • Generate exactly one sentence for the short summary

  • Produce exactly one paragraph for the detailed description

The placeholder acts as a mini-rule.

This is structured behavioral programming through text.

Advanced Placeholder Example

You could write:

SHORT_DESCRIPTION_OF_ROLE_BEFORE_1800_IN_ONE_PARAGRAPH

The model interprets it as:

  • Extract role information

  • Restrict it to pre-1800 period

  • Present it in one paragraph

This is more than formatting — it is structured knowledge extraction.

Why This Is Powerful

The Template Pattern allows you to:

  • Strictly control format

  • Extract information from large texts

  • Impose constraints

  • Encode complex rules inside placeholders

If the template is clear, you receive exactly the format you need — without unnecessary text or creative deviation.

Formal Structure of the Template Pattern

To implement this pattern, your prompt should include:

  1. I will provide a template for your output.

  2. X is my placeholder for content.

  3. Place the generated output inside the specified placeholders.

  4. Preserve formatting and overall structure.

How to Define Placeholders

You can mark placeholders in different ways:

  • <FULL_NAME>

  • POSITION_TITLE

  • {COMPANY_NAME}

  • ROLE_DESCRIPTION

The key is clarity.

Example template:

Hello, <FULL_NAME>. You work at <COMPANY_NAME> as a <POSITION_TITLE>.

Or a simpler structure:

NAME, POSITION, COMPANY

The model understands these are variables to be replaced.

Example 1: Workout Template

Create a strength training workout with complementary exercises.
I will provide a template.

WORDS IN ALL CAPS are placeholders.
Insert data into those placeholders and preserve formatting.

Template:

EXERCISE NAME, REPS AND SETS, TARGET MUSCLES, DIFFICULTY 1–5, TECHNIQUE NOTES

Original dishes were culturally specific (syrniki, borscht, Olivier salad).
For an international audience, I replaced them with globally recognizable dishes.

Please create a shopping list to cook from scratch:

  • Classic potato salad

Insert content into the placeholders and preserve the structure.
