Menu Actions Pattern

When we use software applications, they usually include a menu of actions.
For example, a text editor has:

  • File → Open

  • File → Save

  • Edit → Find and Replace

This model is powerful because instead of rewriting instructions every time, we simply select an action from a predefined list.

The same principle can be applied to prompt engineering.

In long conversations with a language model, it can be extremely useful to define a set of reusable commands at the beginning of the dialogue. Instead of repeating complex instructions, we trigger predefined actions using short commands.

Core Idea of the Pattern

We define in advance:

  • What phrase triggers an action

  • What exactly the model must do when it sees that phrase

Each action is essentially a mini-prompt embedded inside the conversation.

Example: Working with an Outline

Assume we are using the Outline Expansion Pattern and building a structured document.

We can define:

When I write:
write <section_number> <number_of_paragraphs>,
you generate text for that section using the specified number of paragraphs.

Here <section_number> and <number_of_paragraphs> are placeholders.

Now I can write:

write 3.2 2

And the model understands:

  • Find section 3.2

  • Write 2 paragraphs about it

  • Then ask what to do next

Adding a Second Action

We can define another rule:

When I write only a section number (e.g., 3.1),
expand that section in outline format.

Now we have two menu actions:

  1. Expand section structure

  2. Write full text for a section

We have effectively created a mini-program inside a conversation.

Why This Is Powerful

1. Repeated Tasks

If you regularly:

  • Draft content based on structure

  • Reformat responses

  • Rewrite text in specific styles

You can define reusable commands once and call them anytime.

2. Efficiency

Instead of rewriting detailed instructions repeatedly, you use short command triggers.

This reduces:

  • Cognitive load

  • Prompt length

  • Repetition errors

3. Team Standardization

In organizations, this pattern allows teams to:

  • Define standard action commands

  • Embed best practices into those commands

  • Share reusable prompt systems

  • Ensure consistent output formatting

It becomes a lightweight workflow system — without writing code.

Example: Shopping Assistant Menu

We can define:

  • When I write: “add PRODUCT”, add PRODUCT to my list and update total cost.

  • When I write: “remove PRODUCT”, remove it and update total cost.

  • When I write: “optimize”, suggest cheaper alternatives.

  • At the end, ask me what action to perform next.

Now the conversation behaves like an interactive tool.

Key Takeaway

The Menu Actions Pattern allows you to:

  • Reuse prompts

  • Automate repeated workflows

  • Build command-based interaction systems

  • Turn conversations into structured operational environments

You are essentially programming behavior — without writing code.
