The Step-by-Step Completion Pattern

Sometimes, when working with a Large Language Model, we only know part of the solution.

We have a task. We understand certain steps that must be included. But we don’t see the full picture. We want the model to help us “complete” the missing parts — essentially finishing the instruction.

This is exactly what the step-by-step completion pattern is designed for.

We define:

  • The goal

  • The known elements that must be included

  • The gaps

Then we ask the model to fill in the missing steps.

This pattern allows the model to:

  • Extend a partially defined process

  • Connect known elements into a coherent sequence

  • Complete missing parts of a procedure

Importantly, the model understands intuitive markers for “gaps,” even if we describe them informally.

Example: Adding a Feature to a Travel Planning App

Let’s return to our travel planning application.

Suppose we introduce a new feature:

I will provide a starting point and a destination, and you generate a complete list of stops, including locations between them.

What are we doing?

We provide an incomplete instruction.

We know:

  • Where the trip starts

  • Where it ends

But we do not define what happens in between.

For example:

Start: New York
End: Washington, D.C.

The model may logically infer intermediate stops such as:

  • Philadelphia

  • Baltimore

It completes the missing part of the route based on geographic logic.

Combining with a Custom Route Format

Earlier, we defined a custom route format. Now we combine both patterns.

Suppose we write:

New York,0 → ... → ... → Miami,2

What does this mean?

  • Start in New York (0 days stay)

  • End in Miami (2 days stay)

  • Two unknown stops in between

  • The model must fill them

The ellipsis (…) indicates a gap.

The model might suggest:

  • Washington, D.C.

  • Savannah

Both are reasonable stops on a road trip along the East Coast.

It respects all constraints:

  • Start defined

  • End defined

  • Number of intermediate stops defined

  • Gaps must be filled

That is the step-by-step completion pattern in action.

A More Open-Ended Example

Now consider a less structured prompt:

Seattle,1 → .......... → San Diego,3

Here:

  • Start: Seattle (1 day)

  • End: San Diego (3 days)

  • The middle section is intentionally vague

The model interprets this as a long road trip down the West Coast and may suggest:

  • Portland

  • San Francisco

  • Los Angeles

Notice how the model infers context. A longer gap suggests a longer journey, so it proposes multiple intermediate stops.

How the Pattern Works

You:

  1. Define the goal

  2. Specify required elements

  3. Mark missing sections

  4. Ask the model to complete the structure

Gaps can be indicated using:

  • Ellipses (…)

  • Placeholder tokens

  • Custom markers

  • Any symbolic notation

The model interprets these as “missing steps” and completes them logically.

Where This Pattern Is Powerful

This approach is useful for:

  • Designing processes

  • Building algorithms

  • Planning routes

  • Structuring instructions

  • Completing workflows

Whenever you know part of a solution but not the entire sequence, the step-by-step completion pattern helps transform fragments into a coherent structure.


Structure of the Step-by-Step Completion Pattern

To use this pattern effectively, your prompt should include the following core elements:

  • I want to achieve X

  • I know that I need to complete steps A, B, C

  • Provide the full sequence of steps

  • Fill in all missing steps

  • (Optional) Identify unnecessary or redundant steps

You replace “X” with a specific goal.
Then you list steps A, B, C — the elements you already know must be included in the final plan or “recipe.”

The model’s role is to complete the structure while respecting your constraints.

Example (Adapted)

I want to drive from New York to Boston.
I know that I want to stop in New Haven along the way and that I do not want to drive more than 300 miles per day.
Provide the full sequence of steps.
Fill in all missing steps.

Here’s what this structure communicates:

  • The goal (drive from New York to Boston)

  • A required intermediate stop (New Haven)

  • A constraint (maximum daily driving distance)

  • A request for a complete plan

The model must:

  • Respect the required stop

  • Respect the distance limitation

  • Add logical intermediate steps

  • Structure the route coherently

This is the step-by-step completion pattern in its formal form.
