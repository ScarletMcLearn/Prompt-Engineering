You may have already noticed that throughout the course, I’ve been combining different prompt patterns. This is one of the most important principles if you want to design advanced, high-quality prompts.

Instead of asking, “Which single pattern should I use?”, a better question is:

  • Which pattern solves which part of the problem?

  • How can I combine them into a coherent system?

We’ve already seen examples of this in practice. For instance, when we combined:

  • The Ask for Input Pattern

  • The Alternative Approaches Pattern

The Ask for Input Pattern helps pause the model and return control to the user.
The Alternative Approaches Pattern encourages the model to generate multiple possible solutions.

When combined, they create a much more powerful prompt than either pattern alone.

That is how strong prompts are built — not through isolated techniques, but through structured combinations.

Thinking Structurally About Complex Prompts

When you face a new, complex task, ask yourself:

  • What components does this problem consist of?

  • Is there a pattern that addresses each component?

  • Can I apply multiple patterns at once?

  • How can I minimize the part where I have to “invent from scratch”?

The idea is to use patterns as building blocks.

At first, this may seem more complicated. But in reality, you are still:

  • Writing in natural language

  • Giving instructions

  • Describing the desired outcome

The difference is that now you are doing it deliberately and systematically.

Combining patterns is essentially a higher level of natural-language programming. You are assembling known instruction structures into a more advanced behavioral system.

Practical Combination Strategy

In practice, combining patterns is often straightforward:

  • Take the required structural elements from one pattern.

  • Add the required structural elements from another.

  • Make sure they are logically ordered.

For example:

In the Ask for Input Pattern, the request for input should almost always appear at the end of the prompt. Otherwise, the model may continue generating content instead of waiting.

Sometimes the sequence requires fine-tuning. But most of the time, effective combination is simply about careful structural assembly.

Core Principle

When designing your own prompts, do not think:

“Which one pattern should I use?”

Instead think:

“Which combination of patterns will produce the behavior I want?”

That is the moment when you move from asking the model questions to intentionally shaping its reasoning framework.
